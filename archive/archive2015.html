December 6, 2015

HTML and CSS overhauls underway in a probably hopeless attempt to make my website friendlier on mobile.

Phase 2 of the HTML/CSS Upgrade Project involves standardizing initial widths for viewports across the site, as well as transferring the main body div CSS to a spreadsheet, where widths will be made relative rather than static. This phase also includes ditching of fixed-width in-line CSS and things that are not actually used but have a negative impact when people visit the site. The STTJ subsite will not be affected by this.

Phase 3 of the HTML/CSS Upgrade Project involves changing CSS via @media queries in order to dramatically improve the quality of menus and other floating divs for a better user experience.

Phase 4 of the HTML/CSS Upgrade Project involves changing fixed width images to work well on mobile via scaling

Here are today's updates:

  • Top level pages upgraded to new HTML/CSS standards [Phase 2]
  • ./projects upgraded to new HTML/CSS standards [Phase 2]
  • ./ph3tutorials upgraded to new HTML/CSS standards [Phase 2]
  • ./classresources upgraded to new HTML/CSS standards [Phase 2]
  • All menus now use @media queries to dynamically shift locations if screen width < 480px [Phase 3]
  • All images should now scale to the browser width if below max size [Phase 4]
  • SeitenTouji subsite should be OK in the mobile department now, with a few exceptions, mainly the site logo scaling weird.
